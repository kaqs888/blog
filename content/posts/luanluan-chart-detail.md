# 一、订单簿深度图

## 1.1 图表结构

```
                    ┌─────────────────────────────────────────┐
                    │               订单簿深度图                │
                    │                                         │
  价格（纵轴）       │     卖方挂单 ← ████████  绿色/红色     │
                    │                █████                    │
       $66,500 ──  │              ██                         │
       $66,000 ──  │  当前价格 ────────────── ← 中间分界线    │
       $65,500 ──  │              ███                        │
                    │            ██████████  → 买方挂单 蓝色  │
                    │          ████████████████  ← 挂单墙！   │
                    │                                         │
                    └─────────────────────────────────────────┘
                              挂单数量（横轴）
```

## 1.2 各元素含义

| 元素 | 颜色 | 含义 |
|------|------|------|
| 买方挂单 | 通常为绿色或蓝色 | 价格下方等待买入的订单 |
| 卖方挂单 | 通常为红色 | 价格上方等待卖出的订单 |
| 挂单墙 | 某一价位的长条 | 该价位堆积了大量订单，形成支撑/阻力 |
| 中心线 | 当前市价 | 买卖双方的分界线 |

## 1.3 如何读取信号

**看多信号：**
- 下方买方挂单明显厚于上方卖方挂单
- 出现大额买方挂单墙
- 上方没有明显的卖方压力

**看空信号：**
- 上方卖方挂单明显厚于下方买方挂单
- 上方出现大额卖方挂单墙
- 下方支撑单薄或逐渐被撤走

**水博乱乱的典型分析：**
> "目前 #BTC 订单簿深度来看... 上面合约和现货都没有显示出很大的挂单压力... 看来在利好走向的消息面前... 资金也没有过多的想在上面出货... 这是利好多头的现象。"

## 1.4 分交易所的订单簿观察

水博乱乱会**同时观察多个交易所**的订单簿：

```
┌──────────────────────────────────────────────┐
│  交易所对比表                                  │
│                                              │
│  Coinbase 现货：  买方 ████████████████████   │ ← 大户/机构
│  Binance 合约：   买方 ████████              │ ← 散户/短线
│  Binance 现货：   买方 ██████                │
│  Bybit 合约：     买方 ███                   │ ← 散户
│                                              │
│  → Coinbase 买方远强于其他 = 真实需求强       │
└──────────────────────────────────────────────┘
```

**重点关注：**
- Coinbase 现货大额挂单 → 机构/大户行为，比较可靠
- 合约挂单远大于现货 → 杠杆投机，不可靠
- 各所挂单方向一致 → 信号更强

# 二、蓝色信号图

## 2.1 图表结构

蓝色信号出现在订单簿差值分析中：

```
┌──────────────────────────────────────────────┐
│                                              │
│  价格走势 ─────────────────────              │
│                         ↓                    │
│                    价格跌到此处              │
│                         │                    │
│  订单簿差值 ──── 5% 以上差值 ───→ 🔵蓝色信号 │
│                                              │
│  买方挂单 ██████████████████████████████     │
│  卖方挂单 ████████                           │
│  差值 = 买方 >> 卖方 → 触发蓝色阈值         │
│                                              │
└──────────────────────────────────────────────┘
```

## 2.2 信号读取方法

1. **出现蓝色信号** → 下方买方挂单极其厚实
2. **中线做多参考** → 胜率 85%+
3. **配合蓝色信号出现时的价格位置** → 越低越有效
4. **确认现货是否参与** → 纯合约挂单可信度低

## 2.3 注意事项

⚠️ 蓝色信号不是"必涨"信号：
- 需要配合其他指标确认
- 在极端行情中可能失效
- 注意区分真实挂单和 Spoofing

# 三、CVD（累积成交量差值）图

## 3.1 图表结构

```
┌──────────────────────────────────────────────┐
│  主图：BTC 价格 K 线                          │
│  ╱╲  ╱╲╱╲   ╱╲                              │
│ ╱  ╲╱    ╲╱╱  ╲                             │
│                  ╲╱╲                          │
├──────────────────────────────────────────────┤
│  副图：CVD 曲线                               │
│                                              │
│  +2b ──────────────                          │
│  +1b ─  ╱╲                                  │
│   0  ─╱  ╲       ←  零轴（买卖力量平衡）     │
│  -1b ─     ╲╱╲                              │
│  -2b ─       ╲   ← 反转阈值！                │
│  -3b ─        ╲  ← 力竭阈值（极端）          │
│                                              │
└──────────────────────────────────────────────┘
     横轴：时间
```

## 3.2 各元素含义

| 元素 | 含义 |
|------|------|
| CVD 曲线上升 | 主动买入力量 > 主动卖出力量 |
| CVD 曲线下降 | 主动卖出力量 > 主动买入力量 |
| CVD = 0 附近 | 买卖力量均衡 |
| CVD 达到 -2b | 区间震荡中可能出现反转 |
| CVD 达到 -3b~-4b | 趋势行情中空头力量力竭 |

## 3.3 价格与 CVD 的背离

| 情况 | 含义 | 操作 |
|------|------|------|
| 价格跌，CVD 跌 | 正常下跌 | 等待力竭 |
| 价格跌，CVD 不跌 | 卖方力量减弱 | 准备反弹 |
| 价格涨，CVD 涨 | 正常上涨 | 跟随趋势 |
| 价格涨，CVD 不涨 | 买方力量虚弱 | 警惕回调 |

## 3.4 CVD 重置的观察方法

```
CVD 重置示意：

CVD    ╱╲
      ╱  ╲         ╱╲
     ╱    ╲       ╱  ╲  ← CVD 回到起始水平 = "重置"
    ╱      ╲     ╱    ╲
起点        ╲   ╱      ╲  ← 如果继续跌 = 新的方向
             ╲╱        归位 → 止盈信号
```

# 四、Volume Profile（成交量分布图）

## 4.1 图表结构

```
┌──────────────────────────────────────────────┐
│  BTC 价格走势                                 │
│                                    Volume     │
│  $70,000 ──  ████                 Profile     │
│  $69,000 ──  █████████                       │
│  $68,000 ──  ████████████████████████ ← POC  │
│  $67,000 ──  ████████████████                │
│  $66,000 ──  █████████                       │
│  $65,000 ──  ████                            │
│                                              │
│  ┌───────────────────────────────┐           │
│  │ VAH = $69,500                │           │
│  │ POC = $68,000  ← 最大成交量   │           │
│  │ VAL = $66,500                │           │
│  └───────────────────────────────┘           │
│                                              │
│  VAH 以上 = 价格偏高区                        │
│  VAL 以下 = 价格偏低区                        │
│  VAL-VAH = 价值区（70% 成交量）               │
└──────────────────────────────────────────────┘
```

## 4.2 如何读取

| 区域 | 价格位置 | 交易含义 |
|------|---------|---------|
| POC 上方 | 偏高 | 关注卖方力量 |
| POC 附近 | 公允价格 | 可能震荡整理 |
| POC 下方 | 偏低 | 关注买方力量 |
| VAH 以上 | 明显偏高 | 看空或谨慎 |
| VAL 以下 | 明显偏低 | 看多或寻底 |

## 4.3 成交量缺口

当两个密集成交区之间有一段"成交量稀薄"的区域，价格经过这里时会**快速穿越**（因为缺少买卖对手）。

# 五、AVWAP 锚定线图

## 5.1 图表结构

```
┌──────────────────────────────────────────────┐
│  BTC 价格走势                                 │
│                                              │
│            ╱╲     ╱╲                          │
│           ╱  ╲   ╱  ╲                        │
│  ────────╱────╲─╱────╲──── AVWAP (从高点锚定) │
│         ╱      ╲      ╲                      │
│   ╱╲   ╱                ╲                    │
│  ╱  ╲ ╱                  ╲                   │
│ ╱    X  ← 锚点（暴跌低点）                    │
│ ────────────────────────── AVWAP (从低点锚定) │
│                                              │
│  两条 AVWAP 收敛 → 即将出方向！               │
└──────────────────────────────────────────────┘
```

## 5.2 各元素含义

| 元素 | 含义 |
|------|------|
| AVWAP 线（从低点锚定）| 从暴跌低点开始的平均持仓成本，通常作为支撑 |
| AVWAP 线（从高点锚定）| 从冲高高点开始的平均持仓成本，通常作为阻力 |
| 两线收敛 | 即将选择方向（突破或跌破）|
| 价格在 AVWAP 上方 | 持有该锚点成本的交易者处于盈利状态 |
| 价格在 AVWAP 下方 | 持有该锚点成本的交易者处于亏损状态 |

## 5.3 典型使用方式

- **止盈目标**：从入场点画 AVWAP，作为目标价位
- **支撑/阻力**：从关键事件点画 AVWAP
- **收敛判断**：当多条 AVWAP 收敛 → 大行情前兆

# 六、OI（持仓量）图

## 6.1 图表结构

```
┌──────────────────────────────────────────────┐
│  主图：BTC 价格                               │
│  ╱╲  ╱╲╱╲   ╱╲                              │
│ ╱  ╲╱    ╲╱╱  ╲╱                            │
├──────────────────────────────────────────────┤
│  副图：未平仓合约 (OI)                        │
│           ╱╲                                 │
│      ╱╲  ╱  ╲  ← OI 历史高位                 │
│     ╱  ╲╱    ╲                               │
│    ╱          ╲                              │
│   ╱            ╲  ← OI 回落到低位             │
│  ╱              ╲                            │
│                   → 多空都撤离，等待新方向      │
└──────────────────────────────────────────────┘
```

## 6.2 关键判断

| OI 状态 | 波动性 | 驱动力 |
|---------|--------|--------|
| OI 高位 | 大 | 合约驱动（杠杆多） |
| OI 低位 | 小 | 现货驱动（更健康） |
| OI 快速增加 | 即将增大 | 新资金入场 |
| OI 快速减少 | 可能增大 | 强制平仓（爆仓） |

# 七、多交易所对比图

## 7.1 不同交易所的成交特征

水博乱乱经常对比不同交易所的行为差异：

```
┌──────────────────────────────────────────────┐
│  多头入场分布                                  │
│                                              │
│  Binance:  ████████████████  (800 BTC)       │
│  Bybit:    ████████  (400 BTC)               │
│  OKX:      ██████  (300 BTC)                 │
│  Coinbase: ██  (100 BTC 现货)                 │
│                                              │
│  解读：合约为主、现货参与少 → 信号偏弱         │
├──────────────────────────────────────────────┤
│  空头入场分布                                  │
│                                              │
│  Binance:  ████████████████████ (1000 BTC)   │
│  Bybit:    ██████████████  (700 BTC)         │
│  OKX:      ████████  (400 BTC)               │
│                                              │
│  空头力量 > 多头 → 短期偏空                    │
└──────────────────────────────────────────────┘
```

## 7.2 关键观察点

- **Coinbase 现货大买单** → 美国机构/大户在买，比较有分量
- **Bybit 散户大量做多** → 常被反向收割，反向参考
- **各所方向一致** → 信号强度高
- **各所方向分歧** → 市场混沌，观望为宜

# 八、综合图表阅读法

## 8.1 水博乱乱的日常看图顺序

```
Step 1: 订单簿 → 确定上方压力和下方支撑位
Step 2: CVD → 确定当前买卖力量对比和趋势
Step 3: Volume Profile → 确定 POC/VAH/VAL 位置
Step 4: AVWAP → 锚定关键成本线
Step 5: OI → 判断市场活跃度和资金状况
Step 6: 分所对比 → 区分机构和散户行为
Step 7: 综合判断 → 是否满足入场模型
```

## 8.2 信号强度评估

| 强度 | 条件 | 操作 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 蓝色信号 + CVD力竭 + 现货大买 + 关键支撑位 | 较大仓位入场 |
| ⭐⭐⭐⭐ | 入场模型触发 + 3个以上指标确认 | 正常仓位入场 |
| ⭐⭐⭐ | 入场模型触发 + 2个指标确认 | 小仓位试探 |
| ⭐⭐ | 部分信号但缺少关键确认 | 观望等待 |
| ⭐ | 信号矛盾或混沌 | 不操作 |
